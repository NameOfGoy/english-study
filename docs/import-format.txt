==============================================
单词/短语 批量导入文件 - 使用说明
==============================================

文件格式: 纯文本 (.txt), 一行一个词条.
字符编码: UTF-8 (含中文标签时务必)
最大行数: 没硬上限, 一行一个词

----------------------------------------------
1. 最简形态 - 无标签
----------------------------------------------

apple
banana
go through

==> 直接导入 3 个词条, 没有任何标签.
    含空格的视为短语 (如 "go through"), 否则视为单词.


----------------------------------------------
2. 标签段 - 给一批词打上同一个标签
----------------------------------------------

[重点词汇]
cat
dog
elephant

==> "cat" / "dog" / "elephant" 都会被打上 "重点词汇" 标签.
    标签若不存在, 会自动建为你自己的"用户标签" (不会自动建系统标签).


----------------------------------------------
3. 切换标签 - 一个文件分多段
----------------------------------------------

[考试词汇]
abundant
benevolent
[日常用词]
fridge
microwave

==> "abundant" / "benevolent" 打 "考试词汇"
    "fridge" / "microwave" 打 "日常用词"
    切换是隐式的: 看到 [日常用词] 就自动结束 [考试词汇] 段.


----------------------------------------------
4. 显式结束 - [---]
----------------------------------------------

[高频词]
the
and
[---]
frog
mountain

==> "the" / "and" 打 "高频词"
    "frog" / "mountain" 没有任何标签 (因为 [---] 关掉了"高频词")

    注意: 终止符必须严格写成 [---] (3 个英文短横线, 区分大小写).
    [--]   ← 不是终止符, 当成标签名 "--" 处理
    [----] ← 不是终止符, 当成标签名 "----" 处理
    [---  ] ← 含空格也不是


----------------------------------------------
5. 行末内联标签 - 给单个词条挂多个标签
----------------------------------------------

格式: 词条文本 后面紧跟一个或多个 [tagN], 中间可有/可无空格.
解析器从行末往前 peel [xxx] 标记, 直到剥不动为止, 剩下的就是词条.

apple [重点词汇]
banana [考试词汇] [日常用词]
look up [重点词汇]
cat[t1][t2][t3]       ← 不带空格紧贴也行

==> 处理结果:
    apple        (重点词汇)
    banana       (考试词汇, 日常用词)
    look up      (重点词汇) — 短语含空格也能识别
    cat          (t1, t2, t3) — 自动建用户级 t1/t2/t3

注意:
* 同一词条的标签按"出现顺序"应用, 去重 (大小写敏感).
* 不闭合的 [bad 不剥, 留在词条文本里 → 词条变 "apple [bad", 导入会失败.
* 内嵌 ] 会让 peel 提前止损, 整行不剥 → 词条保留原样.


----------------------------------------------
6. 段标签 + 内联标签同时用
----------------------------------------------

[考试词汇]              ← 进入段
abundant                ← 仅段标签: (考试词汇)
benevolent [重点]       ← 段 + 内联: (考试词汇, 重点)
benevolent [考试词汇]   ← 内联与段同名, 去重后只挂一次
[---]                   ← 关段
fridge [日常用词]       ← 段已关, 仅内联: (日常用词)

==> 段标签放最前, 内联按顺序追加, 整体去重.


----------------------------------------------
7. 混合示例 - 一个完整文件长什么样
----------------------------------------------

apple
banana [重点]
[考试词汇]
elaborate
exquisite [高频] [写作]
[日常用词]
fridge
look up [口语]
[---]
random_word_no_tag
[新自定义标签]
something_new [新标签2]

==> 处理结果:
    apple                  (无标签)
    banana                 (重点)
    elaborate              (考试词汇)
    exquisite              (考试词汇, 高频, 写作)
    fridge                 (日常用词)
    look up                (日常用词, 口语) — 短语
    random_word_no_tag     (无标签, 因为 [---] 关掉了段)
    something_new          (新自定义标签, 新标签2) — 都会自动建为用户级


----------------------------------------------
8. 标签查找规则 - 系统标签 vs 用户标签同名时
----------------------------------------------

标签按"字符串全等匹配", 大小写敏感.
同名优先级 (导入解析 [标签名] 时):

   ① 系统标签 (所有用户共享, 不可被普通用户编辑/删除)
   ② 你自己的用户标签
   ③ 都没有 ==> 自动新建为"你的用户标签" (永远不会自动建系统标签)

例:
  系统里有标签 "重点词汇" (系统级);
  你自己也建过一个 "重点词汇" (用户级, 颜色不同).

  导入文件写 [重点词汇] 时, 词条会被打上**系统标签**, 不是你自建那个.
  原因: 系统标签共享, 之后分享给别人也能复用; 用户标签是私货.

  如果你确实想用自己那个, 就先把自建那个重命名 (比如 "重点词汇v2"),
  再在导入里写 [重点词汇v2].

注: 这条规则只影响导入解析. 词典页手动给词打标签时, 你看到的是
    全部标签列表 (系统+自己), 自己选哪个就是哪个, 没有自动优先级.


----------------------------------------------
9. 容易踩坑的点
----------------------------------------------

* 空行会被忽略, 不影响标签段的状态.
* 行首尾的空格会被 trim. ("  apple  " 会变成 "apple")
* 标签段 (单独成行的 [tag]) 一次只能激活一个; 但行末内联 [tag1] [tag2] 可以多个.
  想给同一个词打多个标签, 用行末内联格式: apple [t1] [t2]
* []  (空中括号) 不是 marker, 会当作普通词条 "[]" 处理 (然后导入失败).
* [abc  (没有闭合方括号) 也不是 marker, 当词条处理.
* 标签名长度限制 20 个字符. 太长会创建失败 (不影响后续词条导入).
* 一个文件内同一标签名重复出现, 只在第一次"找/建"标签, 后面复用同一个 tag_id.


----------------------------------------------
10. 短语 vs 单词的自动识别
----------------------------------------------

* 不含空格, 全部 a-z 字母 ==> 单词
* 含空格, 或含非字母字符 ==> 短语

例:
  apple       => 单词
  go through  => 短语
  break-up    => 短语 (含非字母 '-')


----------------------------------------------
11. 处理时机
----------------------------------------------

导入是异步的: 提交后立刻返回任务 ID, 实际逐条添加在后台跑.
可以在"我的 -> 导入历史"里看进度与失败列表.
